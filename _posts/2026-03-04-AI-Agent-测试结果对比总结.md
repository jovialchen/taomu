## 📋 测试概览

本次测试对比了三种 AI Agent 工具（Claude Code、OpenCode、DeepAgents）与不同模型（GLM-5、GPT-5.2、Claude 4.6）的交互表现。

---

## 🔍 Agent 核心发现

> 注明：所有对比基于相同模型：glm-5

### 1. 提问方式差异

| Agent 工具 | 提问策略 |
|------------|----------|
| **Claude Code** | 一个问题一个问题单独问 |
| **OpenCode** | 多个问题一起问 |
| **DeepAgents** | 多个问题一起问 |

### 2. Phase 0 问题数量对比

| Agent 工具 | Phase 0 问题数量 |
|------------|------------------|
| **Claude Code** | 4 个问题，提问方向会跑偏，不是所有问题都是一开始提出的问题 |
| **OpenCode** | 1 个问题，没有提问全部一开始的提出的问题 |
| **DeepAgents** | 1 个问题，没有提问全部一开始的提出的问题 |

### 3. 输出文件数量

| Agent 工具 | 输出文件数量 |
|------------|--------------|
| **DeepAgents** | 单个输出文件，没有完全输出所有要求的文件 |
| **Claude Code** | 多个输出文件，输出了所有要求的文件 |
| **OpenCode** | 多个输出文件，输出了所有要求的文件 |

---

## 🧠 模型行为分析

> 注明：所有对比基于相同 agent 框架：deepagents

### GLM-5

> ⚠️ **主要问题**：列举的问题和实际问的问题会跑偏

**对比**：Claude 4.6 也存在此问题，但相比之下表现略微更加稳定一致。

---

## 🤖 Claude 4.6 + DeepAgents 深度观察

### 行为特征

1. **Phase 0 前置工作**：在 phase0 之前会做很多准备工作
2. **无障碍生成 ToDo List**：对比之下，GLM 只有一定概率下会生成 ToDo List
3. **子智能体拆分**：主动拆分子智能体进行任务分配
4. **问题序号混乱**：
   - 第五问问成了第七问
   - 第六问不存在于初始问题清单
   - 额外生成了第 8 问
